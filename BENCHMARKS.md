# BENCHMARKS

This document defines reproducible benchmark commands and expected result bands for cryptex.

## Reproducible Setup

Use a clean environment and run from repository root:

```bash
uv sync --dev
uv run cryptex train
```

Optional quality gate before benchmarking:

```bash
uvx ruff check --fix
uvx ruff format
uvx ty check
uv run pytest -q
```

## Core Commands

Generate convergence diagnostics JSON:

```bash
uv run cryptex analyse --output .\plots
```

Run substitution benchmark at one length:

```bash
uv run cryptex benchmark --length 120 --trials 3 --success-threshold 0.25 --output .\plots
```

Run phase transition benchmark:

```bash
uv run cryptex phase-transition --trials 3 --output .\plots
```

Sweep multiple lengths (PowerShell):

```powershell
$lengths = 80,120,180,260
foreach ($L in $lengths) {
  uv run cryptex benchmark --length $L --trials 3 --success-threshold 0.25 --output .\plots
}
```

## Expected JSON Outputs

The following files are regenerated by the commands above:

- plots/frequency_comparison.json

## Expected Result Bands

The solver stack is stochastic, so acceptance is by ranges instead of exact values.

| Task | Typical Band | Pass Criterion |
| --- | --- | --- |
| Substitution (>=120 chars) | SER 0.00 to 0.20 | SER <= 0.20 |
| Substitution short text (<=30 chars) | unstable | must not crash |
| Vigenere (~100 to 180 chars) | SER 0.05 to 0.25 | SER <= 0.25 |
| Columnar transposition | near-perfect on clean inputs | SER <= 0.10 |
| Affine exhaustive search | full 312-key sweep | tested_keys = 312 |
| Rail fence search (2 to 10 rails) | low SER with correct rail count often recovered | SER <= 0.20 |
| Playfair (>=100 chars) | moderate to strong recovery depending on key/text | must produce finite score and non-empty plaintext |
| Detector confidence | softmax-calibrated probabilities | scores sum to ~1.0 |

## Published-Literature Comparison Notes

The following references are used for directional comparison, not exact replication:

- Jakobsen (1995) style hill-climbing for monoalphabetic substitution: typically near-perfect recovery for longer ciphertexts; cryptex MCMC + tempering is expected to be in the same quality class on medium and long text.
- Classical Friedman/Kasiski Vigenere analysis: key-length detection quality increases with ciphertext length; cryptex uses IoC + Kasiski + n-gram Caesar scoring and should follow this trend.
- Simulated annealing / MCMC cryptanalysis literature: adaptive temperature schedules and multi-start strategies reduce local minima sensitivity; cryptex includes adaptive cooling, replica exchange, and crossover jumps.

Because corpora, preprocessing, and stopping criteria differ by implementation, compare trends and error bands rather than exact wall-clock numbers.
